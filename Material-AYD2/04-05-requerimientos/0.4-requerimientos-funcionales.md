# Drivers RF (Requerimientos Funcionales)
## Sistema de Gestión de Conocimiento (SGC) - Info Hub

Este documento presenta los drivers de requerimientos funcionales derivados de las descripciones textuales de los casos de uso expandidos del sistema.

Cada driver se identifica con un código **RF-X-Y**, donde:
- **X** corresponde al número del caso de uso expandido
- **Y** es el número secuencial del requerimiento dentro de ese caso de uso

---

## CU1 ALMACENAMIENTO Y CLASIFICACIÓN

### CU1-3 Gestionar Borrador (Ver/Modificar/Eliminar)

**RF-1-3-5**  
**Descripción:** Se debe registrar en la bitácora la modificación realizada con la fecha correspondiente.  
**Actores involucrados:** Sistema

---

### CU1-7 Registrar Cambio

**RF-1-7-1**  
**Descripción:** Toda acción que afecte un activo debe quedar asociada al identificador único del mismo.  
**Actores involucrados:** Sistema

**RF-1-7-2**  
**Descripción:** Se debe dejar constancia del responsable de cada cambio registrado.  
**Actores involucrados:** Sistema

**RF-1-7-7**  
**Descripción:** Se debe guardar el registro completo para auditoría.  
**Actores involucrados:** Sistema

> Nota: este listado incluye solo los RF relacionados con las interfaces del ejemplo de código. El resto se omitió, pero se derivan al igual que estos de los CU expandidos y sus descripciones textuales.
